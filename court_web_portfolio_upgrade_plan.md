# Court-WEB 포트폴리오 업그레이드 문서

## 1. 프로젝트 현재 상태 진단
- 현재 프로젝트는 대한민국 대법원 사이트를 참고한 정적 웹사이트 형태로 보인다.
- 저장소 구조는 `index.html`, `common`, `maincss`, `mainjs`, `sub1~sub5`, `member` 등으로 구성되어 있다.
- GitHub 저장소 설명, 배포 링크, README, 기술 설명이 비어 있어 포트폴리오 전달력이 약하다.
- 화면 텍스트와 정보 구조가 원본 사이트의 인상을 강하게 남기기 때문에, 단순 클론으로 보일 위험이 있다.

## 2. 포트폴리오 관점의 핵심 문제
### 문제 1. 프로젝트 정체성이 약함
- 현재는 "법원 사이트를 따라 만든 페이지"로 보일 가능성이 크다.
- 채용 담당자는 "왜 이 프로젝트를 만들었는가"와 "무엇을 개선했는가"를 보고 싶어한다.

### 문제 2. 클론 느낌이 강함
- 메인 문구, 정보 구성, 공공기관 스타일이 원본 사이트를 강하게 연상시킨다.
- 포트폴리오에서는 "참고" 수준을 넘어서 재해석된 결과물이 필요하다.

### 문제 3. UX 개선 포인트가 드러나지 않음
- 정보가 많고 계층이 깊은 공공 웹사이트 특성상, 사용성 개선이 핵심인데 그 전략이 아직 보이지 않는다.

### 문제 4. README / 배포 / 성과 정리가 없음
- 프로젝트 소개, 기술 스택, 문제 해결 과정, 전후 비교가 보이지 않는다.

## 3. 업그레이드 방향
이 프로젝트는 단순한 "대법원 웹사이트 클론"이 아니라,

**"정보 접근성이 낮은 공공기관 사이트를 현대적 UI/UX로 재설계한 리디자인 프로젝트"**

로 포지셔닝하는 것이 가장 좋다.

## 4. 추천 프로젝트 타이틀
### 선택안 A
**Court Information Platform Redesign**
공공 법률 정보 서비스의 접근성과 탐색성을 개선한 웹 리디자인 프로젝트

### 선택안 B
**Judicial Service Web Renewal**
복잡한 사법부 정보 구조를 사용자 친화적으로 재구성한 프로젝트

### 선택안 C
**Public Service UX Redesign – Court WEB**
공공기관 웹서비스의 정보 탐색 경험을 개선하는 데 초점을 둔 UI/UX 리디자인

최종적으로는 다음 인상을 주어야 한다.

## 13. Court-WEB 메인 페이지 포트폴리오용 개편안

### 13-1. 메인 페이지 목표
기존 메인 페이지는 공공기관 사이트의 분위기는 있으나, 정보 우선순위가 명확하지 않고 사용자가 처음 들어왔을 때 무엇을 해야 하는지 빠르게 파악하기 어렵다.

포트폴리오용 개편안의 목표는 아래와 같다.

- 첫 화면에서 서비스 목적이 즉시 이해되도록 한다.
- 사용자가 자주 찾는 기능을 상단에서 바로 접근할 수 있게 한다.
- 공지 / 재판안내 / 민원 / 기관소개를 시각적으로 분리한다.
- 복잡한 공공 웹 구조를 현대적인 카드형 UI로 정리한다.
- 신뢰감, 접근성, 명확성을 동시에 보여준다.

### 13-2. 메인 페이지 핵심 콘셉트
**Modern Public Service UI**

키워드:
- 신뢰감
- 명확한 정보 위계
- 쉬운 탐색
- 정돈된 공공기관 이미지
- 접근성 강화

### 13-3. 전체 레이아웃 구조

### 13-4. 섹션별 상세 개편안

#### 1) 상단 유틸 바
역할:
- 로그인
- 사이트맵
- EN
- 접근성 도움말

디자인 방향:
- 높이 얇게
- 텍스트 크기 작게
- 헤더보다 덜 강조
- 배경은 아주 연한 회색 또는 화이트

포인트:
- 불필요하게 튀지 않도록 하고 실사용 보조 기능만 배치한다.

---

#### 2) 헤더 / 글로벌 네비게이션
역할:
- 브랜드 로고
- 1차 메뉴
- 검색 버튼

추천 메뉴 구조:
- 재판안내
- 민원안내
- 법원소개
- 정보공개
- 참여마당

개선 포인트:
- 현재 메뉴가 길고 복잡하다면 5개 정도의 대표 카테고리로 압축
- hover 시 2차 메뉴 드롭다운 제공
- 모바일에서는 햄버거 메뉴 전환

디자인 포인트:
- 로고 왼쪽 정렬
- 메뉴는 중앙 또는 우측 정렬
- 검색 아이콘 / 버튼 추가

---

#### 3) 메인 Hero 영역
현재 문제:
- 단순 비주얼과 추상 문구만 있으면 사용 목적이 모호해진다.

개편 방향:
- Hero를 "메시지 + 행동 유도" 구조로 변경
- 너무 큰 이미지 슬라이더보다 정적인 핵심 메시지가 더 적합

예시 문구:
- **국민이 더 쉽게 이해하는 사법 서비스**
- 재판 절차, 민원 정보, 주요 공지사항을 빠르게 확인할 수 있도록 재구성한 공공 법률 정보 서비스

CTA 버튼 예시:
- 재판 절차 안내
- 전자민원 바로가기
- 공지사항 보기

디자인 제안:
- 좌측: 텍스트
- 우측: 법원/문서/공공서비스를 상징하는 일러스트 또는 단정한 이미지
- 배경은 화이트 또는 매우 연한 블루 그라데이션

---

#### 4) 자주 찾는 서비스 바로가기
이 섹션은 메인 페이지에서 가장 중요하다.

구성 예시:
- 재판 절차 안내
- 전자소송/전자민원
- 청사 안내
- 사건 검색
- 공지사항
- 민원 서식 안내

디자인 방식:
- 6개 카드 또는 아이콘 카드
- 3열 2행 또는 6열 단일 행
- 아이콘 + 제목 + 짧은 설명

예시 설명:
- **재판 절차 안내**: 민사·형사·가사 재판 흐름을 쉽게 확인
- **전자민원 서비스**: 온라인으로 필요한 민원 업무를 빠르게 처리
- **청사 안내**: 법원 위치와 방문 정보를 확인

포트폴리오 포인트:
- "사용자가 가장 먼저 찾는 기능을 메인 상단에 재배치했다"는 UX 논리를 보여줄 수 있다.

---

#### 5) 공지 / 보도자료 / 입찰정보 탭 섹션
현재 공공기관 사이트는 정보가 많지만 사용자는 최신 정보만 보고 싶어하는 경우가 많다.

개편 방향:
- 탭 UI 도입
- 기본 탭은 공지사항
- 각 탭당 4~5개 최신 항목 노출
- 더보기 버튼 제공

표현 방식:
- 좌측: 섹션 제목 및 설명
- 우측: 탭 콘텐츠

예시 탭:
- 공지사항
- 보도자료
- 입찰정보

디자인 포인트:
- 날짜 강조
- 제목 hover 상태 명확화
- 더보기 버튼을 일관된 스타일로 통일

---

#### 6) 재판절차 / 민원안내 / 전자서비스 카드 섹션
역할:
- 서비스 성격이 다른 정보들을 구역별로 분리

구성 방식:
3개의 대형 카드
- **재판절차 안내**
- **민원안내**
- **전자서비스**

각 카드 내부:
- 짧은 설명
- 대표 링크 2~3개
- 관련 아이콘 또는 썸네일

예시:
- 재판절차 안내 → 민사재판, 형사재판, 가사재판
- 민원안내 → 증명서 발급, 제출 서류, 방문 민원
- 전자서비스 → 전자소송, 사건조회, 온라인 민원

장점:
- 정보 군집이 명확해져서 사용자가 어디로 들어가야 하는지 직관적으로 알 수 있다.

---

#### 7) 기관 소개 / 사법부 역할 요약 섹션
현재 공공 사이트에서는 기관 소개가 장황한 경우가 많다.

개편 방향:
- 긴 설명보다 핵심 메시지 3개로 요약

예시 구성:
- 공정한 재판
- 국민 접근성 강화
- 투명한 정보 제공

표현 방법:
- 3개 카드 또는 3단 인포 블록
- 각 항목별 짧은 설명

예시 문구:
- **공정한 재판**: 신뢰할 수 있는 재판 절차와 기준 제공
- **국민 접근성 강화**: 누구나 쉽게 찾고 이해할 수 있는 정보 구조 설계
- **투명한 정보 제공**: 공지, 절차, 기관 정보를 명확하게 공개

포트폴리오 관점:
- 단순 소개 텍스트를 읽기 쉬운 구조로 재편한 사례가 된다.

---

#### 8) 통계 또는 신뢰지표 섹션
이 영역은 프로젝트 완성도를 높여준다.

예시 항목:
- 연간 처리 건수
- 전자민원 이용률
- 주요 서비스 만족도
- 전국 법원 네트워크

주의:
- 실제 데이터가 없으면 더미 수치 대신 "예시 지표 영역"으로 구성하거나 제거해도 된다.

디자인 방향:
- 숫자를 크게 강조
- 3~4개 지표 카드
- 단정하고 공식적인 스타일

---

#### 9) 접근성 / 이용안내 배너
공공기관 사이트에 매우 어울리는 영역이다.

배치 예시:
- 키보드 이용 안내
- 화면 낭독기 지원 안내
- 웹 접근성 정책

포인트:
- 단순 배너 1개 또는 2개 카드
- 공공서비스에 대한 신뢰감을 높인다.

---

#### 10) 푸터
포함 요소:
- 기관 주소
- 대표 전화
- 관련 링크
- 개인정보처리방침
- 이용약관
- 저작권 문구

개선 포인트:
- 정보는 많되 정렬은 단정하게
- 푸터 내부 링크 그룹화

---

### 13-5. 메인 페이지 와이어프레임 텍스트 예시

```text
[유틸바] 로그인 | 사이트맵 | EN | 접근성 안내

[헤더]
로고 | 재판안내 | 민원안내 | 법원소개 | 정보공개 | 참여마당 | 검색

[Hero]
국민이 더 쉽게 이해하는 사법 서비스
재판 절차, 민원 정보, 주요 공지사항을 빠르게 확인할 수 있도록 재구성한 공공 법률 정보 서비스
[재판 절차 안내] [전자민원 바로가기] [공지사항 보기]

[자주 찾는 서비스]
재판 절차 안내 | 전자민원 | 청사 안내 | 사건 검색 | 공지사항 | 민원 서식

[공지/보도/입찰 탭]
공지사항 | 보도자료 | 입찰정보
- 최신 항목 리스트

[핵심 서비스 카드 3개]
재판절차 안내
민원안내
전자서비스

[기관 역할 소개]
공정한 재판 | 국민 접근성 강화 | 투명한 정보 제공

[신뢰 지표]
연간 처리 건수 | 전자민원 이용률 | 전국 네트워크 | 서비스 제공 범위

[접근성 배너]
웹 접근성 안내 / 이용자 도움말

[푸터]
기관 정보 / 정책 링크 / 연락처
```

### 13-6. 컬러 및 스타일 제안

#### 컬러
- Primary: 딥 네이비
- Secondary: 차분한 블루
- Background: 화이트 / 라이트 그레이
- Border: 연한 그레이
- Accent: 포인트 블루

#### 타이포
- Hero 제목: 굵고 큼
- 섹션 제목: 중간 크기 / 볼드
- 본문: 16px 전후의 읽기 쉬운 크기
- 설명문: 14~16px

#### UI 요소
- 버튼: 라운드 적당히, 너무 둥글지 않게
- 카드: 얕은 그림자 또는 얇은 보더
- 링크 hover: 색상 변화 + 밑줄 또는 강조

### 13-7. 포트폴리오에서 설명할 개선 포인트
이 개편안을 포트폴리오에 넣을 때는 아래처럼 설명하면 좋다.

1. 기존 메인 페이지는 정보가 많지만 우선순위가 모호했다.
2. 사용자가 자주 찾는 기능을 상단으로 재배치했다.
3. 공지 / 재판 / 민원 / 기관 정보를 명확히 분리했다.
4. 공공기관의 신뢰감은 유지하면서도 현대적인 카드형 UI를 적용했다.
5. 접근성과 반응형 구조를 고려한 레이아웃으로 재설계했다.

### 13-8. 바로 구현할 우선순위
1. Hero 문구 및 CTA 재작성
2. 자주 찾는 서비스 카드 영역 추가
3. 공지사항 탭 UI 추가
4. 재판/민원/전자서비스 카드 3분할 섹션 추가
5. 기관 소개 텍스트 축약 및 재배치
6. 푸터 정리

### 13-9. 내가 추천하는 실제 첫 구현 범위
첫 작업은 메인 페이지 전체를 한 번에 다 바꾸기보다 아래 범위로 시작하는 것이 좋다.

**1차 구현 범위**
- 헤더 유지 보수
- Hero 교체
- 자주 찾는 서비스 섹션 생성
- 공지사항 영역 카드화

이 정도만 바꿔도 화면 인상이 크게 달라지고, 포트폴리오용으로도 훨씬 좋아진다.
