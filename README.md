# Court-WEB

공공기관 웹사이트의 복잡한 정보 구조를 분석하고, 사용자 중심 UI/UX로 재해석한 **포트폴리오 리디자인 프로젝트**입니다.

## 프로젝트 소개

Court-WEB은 실제 대법원 사이트의 정보 구조를 참고하여, 공공기관 웹서비스 특유의 신뢰감은 유지하면서도 더 직관적이고 현대적인 화면으로 개선한 프로젝트입니다.

단순한 클론 제작이 아니라, **정보 탐색 흐름과 가독성을 개선하는 리디자인**에 초점을 두고 진행했습니다.

## 핵심 목표

- 자주 찾는 기능을 더 빠르게 찾을 수 있도록 정보 우선순위 재정리
- 긴 문단형 서브 페이지를 카드형 / 목록형 / 절차형 UI로 개선
- 메인과 서브 페이지 전반의 시각적 일관성 강화
- 반복되는 스타일과 구조를 정리해 유지보수성 향상

## 주요 작업

### 메인 페이지
- Hero 영역 재구성
- 자주 찾는 서비스 상단 배치
- 공지 / 안내 / 소개 섹션의 정보 위계 정리

### 서브 페이지
- **sub2 주요재판 / 주요절차 / 주요판결 / 주요변론** 리디자인
- 표, 검색, 탭, 영상 목록 UI 개선
- 소개형 페이지와 안내형 페이지의 가독성 개선

### 리팩토링
- 반복되는 UI 패턴 정리
- 페이지별 스타일 편차 축소
- 공통 구조와 재사용성 개선 방향 적용

## Before / After

### Before
- 정보가 많고 문단형 나열이 많아 탐색이 어려움
- 페이지별 스타일 일관성이 약함
- 공공기관 사이트 특유의 올드한 인상이 강함

### After
- 사용자 중심으로 정보 구조를 재배치
- 카드형 UI, 정돈된 여백, 명확한 타이포 위계 적용
- 메인과 서브 페이지 전반의 완성도 향상

- CSS3
- JavaScript

- AOS

## 내가 맡은 역할

- 프로젝트 기획 및 리디자인 방향 설정
- 메인 / 서브 페이지 UI 개선
- HTML / CSS 퍼블리싱
- 기존 구조를 유지한 리팩토링
- 포트폴리오용 프로젝트 문서화

## 배운 점

이 프로젝트를 통해 정보가 많은 공공기관 웹사이트를 단순히 꾸미는 것이 아니라, **사용자가 더 쉽게 이해하고 탐색할 수 있도록 구조를 개선하는 과정**이 중요하다는 점을 배웠습니다.

또한 정적 웹사이트에서도 공통 패턴 정리와 UI 일관성 확보가 프로젝트 완성도에 큰 영향을 준다는 점을 경험했습니다.

## 아쉬운 점

- 실제 사용자 테스트 기반 개선은 진행하지 못함
- 일부 공통 구조와 JS는 추가 리팩토링 여지가 있음
- 접근성 / SEO는 추가 개선 가능

## 포트폴리오 포지셔닝

Court-WEB은 단순한 클론 프로젝트가 아니라,

**"공공기관 웹서비스를 분석하고 사용자 경험 중심으로 재구성한 UI/UX 리디자인 프로젝트"**

로 포지셔닝하고 있습니다.
